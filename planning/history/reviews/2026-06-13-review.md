### [SEC] / [STYLE] / [SILENT] hooks/lib/skill-router.ts:74 — ALLOWED_INSTALL_PREFIXES blocks prompt installs
> `ALLOWED_INSTALL_PREFIXES` contains only `~/.claude/skills/` and `~/.claude/agents/`. `commands/loom-library.md` Phase 3 instructions tell the installer to call `validateInstallPath(targetPath)` for every item in `/loom-library use <name>`. Prompts install to `~/.claude/commands/` — every `/loom-library use <prompt-name>` would abort with `SOURCE_VALIDATION_ERROR`.
**Fix:** Either (a) add `~/.claude/commands/` to `ALLOWED_INSTALL_PREFIXES`, OR (b) scope the validateInstallPath call in `commands/loom-library.md` to only `type: skill` and `type: agent` items. Option (a) is simpler and matches the validator script's prefix table. *Found by: code-reviewer, silent-failure-hunter*

### [STYLE] / [SILENT] hooks/lib/wizard-interview.ts:341,383 — description emitted to YAML without escaping
> `generateSkillMdContent` and `generateLibraryYamlEntry` interpolate `answers.description` and trigger globs verbatim into multi-line YAML. A description containing `:`, `#`, `[`, or a newline produces invalid YAML that `js-yaml.safe_load` silently misparses, breaking `detectExistingSkill` and the validator. Only happy-path values tested.
**Fix:** Wrap `description` in YAML double-quotes and escape embedded quotes/backslashes. Reject newlines in description at `ask-description` interview step. Glob entries should also be quoted in `generateLibraryYamlEntry`. *Found by: code-reviewer, silent-failure-hunter, security-reviewer (CWE-116)*

### [ARCH] commands/loom-library.md:72,229 — name regex drift from wizard SLUG_RE
> Inline regex `^[a-zA-Z0-9][a-zA-Z0-9_-]*$` (allows uppercase + underscore) diverges from the wizard's canonical `^[a-z][a-z0-9-]*$` in `hooks/lib/wizard-interview.ts`. **Fix:** Extract a single `validateItemName(name)` in `hooks/lib/skill-router.ts`. Reference it from the markdown the same way `validateInstallPath` is referenced. *Found by: architecture-reviewer*

### [ARCH] / [TEST] hooks/lib/skill-router.ts:65 — BARE_NAME_PRIORITY silently omits `infrastructure`
> `ResourceType` includes `infrastructure` and the validator + markdown treat it as a first-class section, but `BARE_NAME_PRIORITY = ['agent', 'protocol', 'skill', 'prompt']` skips it. Bare-name includes for infrastructure items silently raise `NOT_IN_CATALOG`.
**Fix:** Append `'infrastructure'` to `BARE_NAME_PRIORITY` as lowest priority. Add a test asserting resolution behavior. *Found by: architecture-reviewer, pr-test-analyzer*

### [TEST] scripts/validate-library-catalog.js — no fixture-driven test for new v4 behavior
> Wave 1 + Wave 2 extended the validator to accept `protocol:` / `infrastructure:` prefixes in `requires:` AND to walk typed-object `kit.includes`. Only smoke-tested against the live catalog. A regression that flips one prefix or breaks the typed walker would pass CI.
**Fix:** Add `test/validate-library-catalog.test.ts` with fixtures exercising both branches (protocol/infrastructure prefix; typed-include object; malformed object). *Found by: pr-test-analyzer*

### [SILENT] hooks/lib/library-catalog-migrator.ts:428 — unsafe cast on v3 skills items
> `v3Library.skills` items are cast directly to `ProtocolEntry[]` without field-level validation. Missing `name`/`source`/`description` fields produce malformed v4 entries that fail at read time with opaque errors instead of `MigrationValidationError` at migrate time.
**Fix:** Add field-level validation in the loop; throw `MigrationValidationError` with the index on missing required fields. Mirror existing `validateRepoUrl`/`validateSemver` pattern in the same file. *Found by: silent-failure-hunter*

### [SEC] scripts/validate-library-catalog.js:324 — `source` path not clamped to REPO_ROOT
> `path.resolve(REPO_ROOT, entry.source)` then `fs.existsSync` — no check that the resolved path stays inside `REPO_ROOT`. A crafted `source: "../../.ssh/authorized_keys"` becomes a filesystem existence oracle. Impact is limited (attacker must already write `library.yaml`; existence-only, no read/write) but the fix is a one-liner.
**Fix:** After `path.resolve()`: `if (!sourcePath.startsWith(REPO_ROOT + path.sep)) { errors.push(...); continue; }` *Found by: security-reviewer (CWE-22)*

### [ARCH] commands/loom-library.md:213-220 — heuristic signal cascade re-derived in markdown

### [SILENT] hooks/lib/wizard-interview.ts:3040-3045 — detectExistingSkill brace-balance heuristic over-rejects

**Fix:** Document the known false-positive cases in a comment, OR replace the heuristic with a try/catch around the actual regex scan. *Found by: silent-failure-hunter*

### [ARCH] hooks/lib/skill-router.ts:148 — buildSkillInstallRecord epoch sentinel default

### [CONTRACT] checksums.sha256 — `commands/loom-skill.md` not in manifest

**Fix:** `echo "$(shasum -a 256 commands/loom-skill.md) commands/loom-skill.md" >> checksums.sha256` once, then `scripts/generate-checksums.sh` to normalize. *Found by: plan-compliance-reviewer*

### [SEC] scripts/validate-library-catalog.js:385 — `inc.type` not allowlisted before section lookup
> `inc.type` is interpolated into `${inc.type}s` and used as a Map key. A Map is used so prototype pollution isn't exploitable, but invalid types fall through to a false "name not found" warning instead of a schema error.
